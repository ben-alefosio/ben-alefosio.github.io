TECHNICAL BLOG


Hi reader!
For the past three weeks I've been learning HTML, CSS and Javascript.

These are three languages used in front end web developing to help
the experience/presentation for users.

HTML, short for HyperText Markup Language, is a type of markup language
that helps format text so that it's able to translate through a web browser
like Google Chrome or Firefox. The text that you see on a site
stems from 'elements' that form the HTML, eg. a header or a paragraph.
A family of these various elements collectively help to
build what is essentially the body of the site.

CSS, short for Cascading Style Sheets, I would consider the webpage's clothes.
CSS is a style sheet language, a way of dressing up the text so that
when presented on the site, it looks much more presentable. Think of the
banner, the side bars, the colours, the positioning of different elements
like headers and paragraphs. This is all due to CSS, and a page would look
quite dull and uninteresting without it.

If the first two are the body and the clothes, then JavaScript is obviously
the magical medallion that helps accomplish tasks for the user.

JavaScript is a programming language central to the world wide web much like
HTML and CSS, but it's reach far extends the other two in usability.
Being able to interact with your website is thanks to JS.
Instead of just having content presented, you can, for example, play a game on
a website (thus you can show animations). You can zoom in, you can have a
menu drop down on a fast food website, you can have a timer that rings when
you want it to. If HTML and CSS are a book's words and colors and pictures
then JS is a page that pops up on the book, and then some.

As JS is the more complicated of the three, the way to write it up ( at least
from my experience ) is also much harder. How to get it to work is through executing
statements. You have that medallion but you actually have to speak some magic
words into it to get it to work.
Control Flow is the statement's code (magic words) put in an order
that makes sense enough that it will return with a desired result.
Countless functions are in the background working so that one action
taken has a domino effect on the next, and this may lead to something as simple
as a prompt on a website asking whether you'd like to add more to your cart.

A loop for example is a single statement that will, as the name suggests, repeat
instructions over and over up to the point that you set it to stop.
Consider needing to make a stew for hundreds of people but only having recipe for 10.
Well through a loop, this recipe will run the recipe over 10 times to help
accomodate the hungry mouths.

As JavaScript is multi-faceted it can also be used to make mobile apps.
If you want to create simple apps that show you weather or keep inventory
of the books you have in your home, this can be set up through JS.
